## 1. מטרה ורקע

### בעיה
מערכת של מספר סוכני AI עצמאיים אינה שמישה אם המשתמש צריך לדעת לאיזה סוכן לפנות. כל ממשק ישיר מרובה סוכנים יוצר עומס קוגניטיבי ומגדיל את שיעור השגיאות.

### פתרון
סוכן מנכ"ל יחיד שמשמש נקודת כניסה בלעדית. המשתמש פונה תמיד לאותו סוכן; המנכ"ל מנתב פנימית לסוכן הנכון באופן שקוף.

### מטרה עסקית
לאפשר הפעלת מערכת חמישה סוכנים עם ממשק משתמש פשוט של "סוכן אחד" — ללא ידע נדרש על ארכיטקטורה פנימית.

## 3. דרישות פונקציונליות

### 3.1 קבלת משימה

- **FR-01:** המנכ"ל מקבל **כל** משימה מהמשתמש — אין bypass ישיר לסוכן כפוף.
- **FR-02:** המנכ"ל קורא את ההודעה המלאה לפני כל פעולה.
- **FR-03:** אם המשימה עמומה — המנכ"ל שואל שאלת הבהרה אחת ממוקדת ומחכה לתשובה.

### 3.2 ניתוח וסיווג

- **FR-04:** המנכ"ל מסווג כל משימה לפי דומיין אחד (או יותר) מתוך 4 הדומיינים המוגדרים.
- **FR-05:** אם המשימה חוצה דומיינים — המנכ"ל מתאם ריצה סדרתית (A → B) ומעביר תוצאות ביניים.
- **FR-06:** אם המשימה לא נכנסת לאף דומיין — המנכ"ל מודיע למשתמש ומציע אפשרויות.

### 3.3 ניתוב לסוכן

- **FR-07:** המנכ"ל מפעיל סוכן כפוף דרך **Task tool** בלבד.
- **FR-08:** כל brief לסוכן כולל: תיאור המשימה המלא, הפלט הצפוי, אילוצים, הקשר רלוונטי.
- **FR-09:** משימות מורכבות או בלתי-הפיכות — המנכ"ל מציג את כוונת הניתוב ומבקש אישור לפני ביצוע.

### 3.4 מעקב ואיסוף תוצאות

- **FR-10:** המנכ"ל ממתין לתוצאת הסוכן הכפוף לפני שהוא מדווח למשתמש.
- **FR-11:** בריצה סדרתית — המנכ"ל מעביר את תוצאת הסוכן הראשון לסוכן השני כחלק מה-brief.

### 3.5 תקשורת עם המשתמש

- **FR-12:** ברירת מחדל — עבודה שקטה. המשתמש רואה רק את הסיכום הסופי.
- **FR-13:** הסיכום מכיל: מה נעשה, איזה סוכן עשה זאת, מה התוצאה — ב-2-3 משפטים.
- **FR-14:** שפת התגובה זהה לשפת המשתמש (עברית → עברית, אנגלית → אנגלית).

## 6. לוגיקת ניתוב

```
משימה נכנסת
    │
    ▼
[סיווג דומיין — פעולה אוטונומית, הבהרה רק כמוצא אחרון]
    │ תמונה / ויזואל / עיצוב → Task(יובל, brief)
    │ Agent 2 Domain        → Task(Agent 2, brief)
    │ Agent 3 Domain        → Task(Agent 3, brief)
    │ Agent 4 Domain        → Task(Agent 4, brief)
    │ מרובה                 → Task(Agent A) → תוצאה → Task(Agent B, brief + תוצאה A)
    │ לא מוגדר             → שאלת הבהרה אחת
    ▼
[בדיקת מורכבות]
    │ גבוהה/בלתי-הפיכה → הצג כוונה → המתן אישור → בצע
    │ רגילה → בצע ישירות
    ▼
[קבלת תוצאה]
    ▼
[סיכום למשתמש: 2-3 משפטים]
```

## 8. אינטגרציה עם CLAUDE.md

הסוכן מופעל על ידי Claude Code כ-**sub-agent** אוטומטי בכל שיחה. ה-`description` בקובץ `ceo_agent.md` מגדיר מתי Claude Code מפנה אליו:

```yaml
description: >
  Use this agent as the PRIMARY ENTRY POINT for ALL user requests without exception.
  ...
```

**מה CLAUDE.md צריך להכיל (למפתח לעדכן):**

```markdown
## CEO Agent

All user requests are handled by the CEO Agent first.
The CEO Agent orchestrates the five-agent system and routes
tasks to the appropriate sub-agent automatically.
```
